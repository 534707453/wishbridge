# WishBridge 本地部署说明

## 🚀 快速开始

### 方法一：使用批处理脚本（推荐新手）

1. 双击 `deploy.bat` 文件
2. 等待自动安装依赖并启动服务
3. 服务会在新窗口中打开

### 方法二：使用PowerShell脚本（推荐）

1. 右键点击 `deploy.ps1`，选择"使用PowerShell运行"
2. 或在PowerShell中执行：
   ```powershell
   .\deploy.ps1
   ```
3. 如果提示执行策略问题，先运行：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```

## 📋 前置要求

- Node.js 16+ (已安装)
- npm (随Node.js一起安装)
- Windows 10/11

## 📁 项目结构

```
wishbridge/
├── deploy.ps1          # PowerShell一键部署脚本
├── deploy.bat          # 批处理一键部署脚本
├── 部署说明.md         # 本文档
├── backend/            # 后端服务
│   ├── src/           # 源代码
│   ├── data/          # 数据库文件
│   └── logs/          # 日志文件
└── frontend/          # 前端应用
    ├── src/           # 源代码
    └── dist/          # 构建产物
```

## 🌐 访问地址

- **前端应用**: http://localhost:3000 (或自动分配的端口)
- **后端API**: http://localhost:8080
- **内网穿透**: jp-2.frp.one:35661

## ⚙️ 手动部署步骤

如果需要手动部署，按以下步骤：

### 1. 安装后端依赖
```bash
cd backend
npm install
```

### 2. 安装前端依赖
```bash
cd frontend
npm install
```

### 3. 启动后端服务
```bash
cd backend
npm start
```

### 4. 启动前端服务（新终端）
```bash
cd frontend
npm run dev
```

## 🔧 故障排查

### 端口被占用
- 脚本会自动释放8080端口
- 如遇其他端口冲突，手动关闭相关进程

### 依赖安装失败
```bash
# 清除缓存后重新安装
npm cache clean --force
rmdir /s /q node_modules
npm install
```

### PowerShell执行策略问题
```powershell
# 临时允许脚本执行
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process

# 或永久允许（仅当前用户）
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## 📝 注意事项

1. 首次运行会自动安装依赖，需要联网
2. 脚本会自动检查并释放8080端口
3. 按 Ctrl+C 可停止PowerShell脚本中的服务
4. 关闭窗口可停止批处理脚本启动的服务
